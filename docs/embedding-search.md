# Semantic Search over AI Image Descriptions

Search your photo collection by meaning using embeddings generated from the AI descriptions in `database/mediameta.db`.

## Overview

The `image_description` table contains natural-language descriptions of photos generated by SmolVLM (see `docs/mlx-vlm-usage.md`). By converting these descriptions into embedding vectors, we can find photos that match a search query by semantic similarity rather than exact keyword matching.

**Stack:**
- [llm](https://llm.datasette.io/) - Simon Willison's CLI tool for working with LLMs
- [llm-sentence-transformers](https://github.com/simonw/llm-sentence-transformers) - Plugin for local embedding models
- [sentence-transformers/all-MiniLM-L6-v2](https://huggingface.co/sentence-transformers/all-MiniLM-L6-v2) - Fast, local embedding model (no API key required)

**Inspiration:** [Simon Willison's blog post on embeddings with llm](https://simonwillison.net/2023/Sep/4/llm-embeddings/)

## Prerequisites

- `llm` CLI installed (`pipx install llm` or `brew install llm`)
- Populated `image_description` table in `database/mediameta.db`

## Installation

### Install the sentence-transformers plugin

```bash
llm install llm-sentence-transformers
```

### Register the embedding model

```bash
llm sentence-transformers register all-MiniLM-L6-v2
```

This downloads the model (~90MB) on first use. Verify it's available:

```bash
llm embed-models | grep MiniLM
# sentence-transformers/all-MiniLM-L6-v2
```

## Phase 1: Small Sample Test

Test with a few rows before embedding the full dataset.

### Embed a sample

```bash
llm embed-multi descriptions \
  --attach media database/mediameta.db \
  --sql "SELECT file AS id, description AS content FROM media.image_description WHERE error = 0 LIMIT 5" \
  -d database/embeddings.db \
  -m sentence-transformers/all-MiniLM-L6-v2 \
  --store
```

Flags:
- `--attach media database/mediameta.db` - Attach the source database with alias `media`
- `--sql "..."` - SQL query to select rows (first column = id, second = content)
- `-d database/embeddings.db` - Store embeddings in this database
- `-m sentence-transformers/all-MiniLM-L6-v2` - Embedding model to use
- `--store` - Save the original text alongside the embedding (enables content in search results)

### Verify with a search

```bash
llm similar descriptions -d database/embeddings.db -c "kids playing soccer"
```

Output is newline-delimited JSON sorted by similarity score (highest first):

```json
{"id": "./0/0001b8b0-015a-4eb7-acc4-eefc47d2317e.JPG", "score": 0.48, "content": "...", "metadata": null}
{"id": "./0/00061c11-b28d-44b0-967a-7a53e01e80ea.JPG", "score": 0.44, "content": "...", "metadata": null}
```

Limit results with `-n`:

```bash
llm similar descriptions -d database/embeddings.db -c "snow house winter" -n 3
```

## Phase 2: Full Batch Embedding

Once the sample test looks good, embed all descriptions.

### Remove previous test data

```bash
rm database/embeddings.db
```

### Embed all descriptions

```bash
llm embed-multi descriptions \
  --attach media database/mediameta.db \
  --sql "SELECT file AS id, description AS content FROM media.image_description WHERE error = 0" \
  -d database/embeddings.db \
  -m sentence-transformers/all-MiniLM-L6-v2 \
  --store
```

### Verify row count

```bash
sqlite3 database/embeddings.db "SELECT count(*) FROM embeddings"
```

This should match:

```bash
sqlite3 database/mediameta.db "SELECT count(*) FROM image_description WHERE error = 0"
```

## Searching

### Basic search

```bash
llm similar descriptions -d database/embeddings.db -c "beach sunset"
```

### Limit results

```bash
llm similar descriptions -d database/embeddings.db -c "dog in yard" -n 5
```

### Pipe results through jq for readable output

```bash
llm similar descriptions -d database/embeddings.db -c "mountain hiking" -n 3 \
| jq '{id, score}'
```

### Search without stored content

If you embedded without `--store`, results will have `"content": null`. The `id` field (the filename) is still returned so you can look up the original description:

```bash
sqlite3 database/mediameta.db \
  "SELECT description FROM image_description WHERE file = './0/0001b8b0-015a-4eb7-acc4-eefc47d2317e.JPG'"
```

## Rebuilding Embeddings

To rebuild after adding new descriptions or switching models, delete and re-embed:

```bash
rm database/embeddings.db

llm embed-multi descriptions \
  --attach media database/mediameta.db \
  --sql "SELECT file AS id, description AS content FROM media.image_description WHERE error = 0" \
  -d database/embeddings.db \
  -m sentence-transformers/all-MiniLM-L6-v2 \
  --store
```

To use a different collection name (e.g., to keep multiple model results):

```bash
llm embed-multi descriptions-v2 \
  --attach media database/mediameta.db \
  --sql "SELECT file AS id, description AS content FROM media.image_description WHERE error = 0" \
  -d database/embeddings.db \
  -m sentence-transformers/all-MiniLM-L6-v2 \
  --store
```

Then search against the new collection:

```bash
llm similar descriptions-v2 -d database/embeddings.db -c "query"
```

## Alternative Models

### OpenAI (requires API key)

```bash
llm embed-multi descriptions \
  --attach media database/mediameta.db \
  --sql "SELECT file AS id, description AS content FROM media.image_description WHERE error = 0" \
  -d database/embeddings.db \
  -m 3-small \
  --store
```

### Other sentence-transformers models

```bash
llm sentence-transformers register all-mpnet-base-v2
```

Larger models may produce better results at the cost of speed and disk space.

### Gemini (via llm-gemini plugin)

Install the plugin if not already present:

```bash
llm install llm-gemini
```

Then list available Gemini embedding models:

```bash
llm embed-models | grep -i gemini
```

## Datasette Integration

Add `embeddings.db` to the existing Datasette startup command (see `README.md`). From the `database/` directory:

```bash
cd database
uv run datasette -p 8001 --root \
  --load-extension=spatialite \
  --template-dir ../datasette/templates \
  --plugins-dir=../datasette/plugins \
  -c ../datasette/datasette.yaml \
  mediameta.db embeddings.db
```

This makes the embeddings table browsable at `http://127.0.0.1:8001/embeddings/embeddings`.

For embedding-powered search in the Datasette web UI, install [datasette-llm-embed](https://github.com/simonw/datasette-llm-embed):

```bash
datasette install datasette-llm-embed
```

Because `embeddings.db` and `mediameta.db` are served together, you can write cross-database SQL in Datasette's query editor to join search results back to the source descriptions or EXIF data.

## File Locations

| File | Purpose |
|------|---------|
| `database/mediameta.db` | Source descriptions in `image_description` table |
| `database/embeddings.db` | Generated embeddings (gitignored via `database/`) |
